Casework responsive design

This is to explore, document, show, share, aid discussion of what the fundamental responsive framework should achieve.

What do we know about caseworkers kit

What do we know, or can, or should assume - what constraints (enabling and restrictive) should we consider.

  • the predominant kit is - a regular desk with a base station and 2 monitors
  • desktop monitor default resolution in landscape orientation is 1680 x 1050
  • most users have both monitors in landscape orientation
  • some users have one monitor (the one for reading documents) in portrait resolution
  • desktop monitor default resolution in portrait orientation is 1050 x 1680
  • There is a display setting in Windows 10 - Custom scaling: Change the size of text, apps and other items - which is set to 175%, set by org policy - So this may affect size fo elements on screen?
  • most caseworkers have the 'stock keyboard and mouse', but there are a variety of mice and to a lesser extend keyboard, plus caseworkers do have a range of other personal assistive physical kit
  • Internet explorer, version ??, is the default browser
  • Internet explorer is set by policy to default to 125% zoom level - dunno if many people have changed this
  • Some - how many ??? - caseworkers have chrome browser??
  • Some caseworkers have the 'casework tools' on the left monitor and the 'documents' on the right monitor (or some tools on one, and some other tools on the other). Some caseworkers have it the opposite way around
  • One monitor - some caseworkers, although we've not seen yet, may have only one monitor? Or might be in a situation with only one monitor? e.g. in a meeting? or when screen sharing, or when meeting the public in the public meeting rooms
  • Some, although we've not seen yet, caseworkers may work on a laptop e.g. as a temporary situation like in a meeting. e.g. on-site going to an ad-hoc meeting, or in a space with no meeting kit.
  • Non-caseworkers may need to show or demonstrate systems (in production or a test region) or prototypes with staff or senior management
  • Laptops - there might be a standard display resolution for our standard kit, but there may be a variety of laptops
  • tablets - we don't think there is a need/situation to do casework on a tablet. However they may be used to demonstrate something rather than do casework on - possibly a casework management dashboard? However technology in actual use changes fast.
  • phones - we don't think there is a need/situation for phones, but could they may be used to demonstrate something on, check in on - possibly a casework management dashboard? So not core casework.

Design Assumptions

  • We should optimise for the most common kit/situations. We probably don't need a mobile first design (but it shouldn't break), but we should ensure system is highly usable on single desktop displays, laptops, and tablets
  • Page template: We need a page template, that meets the initial version of casework systems, but is able to be easily extended
  • Accessibility: We need to not overlook accessibility - build in the structure needed at this page to not exclude by oversight. (page template includes block for skip links)
  • Maximise working space: We need to be able to maximise the screen space available to do casework
  • We need to demonstrate/document the resolution break points, gutter, margins, padding - etc - the rules of the layout
  • We have to think about this structure - how it will work with the components and patterns inside it - but we should document/set/explore the style/layout first
  • Other considerations: background colour
  • copying govuk ideas, and other depts casework/internal system ideas - is the basis, so we reuse, and add to the pool of proven design inside gov.
