This is to explore, document, show, share, aid discussion of what the fundamental responsive framework is for a casework system.

We don't know yet exactly how this system will be - eg we are exploring ideas for 2 or 3 main columns, but we do know some things, and we should explore what that means/might be like.

What do we know about caseworkers kit

What do we know, or can, or should assume - what constraints (enabling and restrictive) should we consider.

  • They are predominantly desktop users - sat a desk with a base station and 2 monitors
  • desktop monitor default resolution in landscape orientation is 1680 x 1050
  • most users have both monitors in landscape orientation
  • some users have one monitor (the one for reading documents) in portrait resolution
  • desktop monitor default resolution in portrait orientation is ???? x ????
  • Internet explorer, version ??, is the default browser
  • Internet explorer is set by policy to default to 125% zoom level - dunno if many people have changed this
  • There maybe display settings - set by org policy - to increase the DPI size? there used to be
  • Some - how many ??? - caseworkers have chrome browser??
  • Some caseworkers have the 'casework tools' on the left monitor and the 'documents' on the right monitor (or some tools on one, and some other tools on the ther). Some caseworkers have it the opposite way around
  • Some, although we've not seen yet, caseworkers may have only one monitor? Or might be in a situation with only one monitor? e.g. in a meeting? or when screen sharing, or when meeting the public in the public meeting rooms
  • Some, although we've not seen yet, caseworkers may work on a laptop e.g. as a temporary situation like in a meeting. e.g. on-site going to an ad-hoc meeting, or in a space with no meeting kit.
  • Non-caseworkers may need to show or demonstrate systems (in production or a test region) or prototypes with staff or senior management

  • we don't know about tablets - they may be used to demonstrate something rather than do casework on - but tech in actual use changes fast.

Design Assumptions

We should optimise for the most common kit/situations. We probably don't need a mobile first design (but it shouldn't break), but we should ensure system is highly usable on single desktop displays, laptops, and tablets

We need a page template, that meets the initial version of casework systems, but is able to be easily extended

We need a grid system

We need to not overlook accessibility - build in the structure needed at this page to not exclude by oversight

We need to be able to maximise the screen space available to do casework

We need to demonstrate/document the resolution break points, gutter, margins, padding - etc - the rules of the layout

We have to think about this structure - how it will work with the components and patterns inside it - but we should document/set/explore the style/layout first

copying govuk ideas, and other depts casework/internal system ideas - is the basis, so we reuse, and add to the pool of proven design inside gov.
